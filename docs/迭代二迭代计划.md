 - 实现kube-proxy来支持service(lab guide 2), service支持两种类型: ClusterIP和NodePort
 - 实现DNS(lab guide 5, 由于service的clusterIp动态分配的, 只实现service不方便测试两个pod的通信)

 - 实现scheduler, 将pod分配到node
